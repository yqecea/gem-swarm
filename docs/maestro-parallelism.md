# Параллельное выполнение в Maestro

Теперь Maestro поддерживает возможность запуска нескольких агентов **с одинаковыми ролями** (например, нескольких `coder` или нескольких `tester`) параллельно. 

Ранее параллельное выполнение могло быть ограничено запуском только различных специализированных агентов. Теперь это ограничение снято: возможность параллельного запуска зависит не от роли агента, а от независимости самих задач.

## Основные принципы нового параллелизма

1. **Независимость задач:** Агенты могут работать параллельно, если они выполняют логически обособленные задачи, не блокирующие друг друга.
2. **Отсутствие пересечений по файлам:** Главное условие для параллельного запуска — каждый агент в рамках параллельного пакета (batch) должен работать со своим собственным, непересекающимся набором файлов. Это предотвращает конфликты при записи и обеспечивает безопасность файловой системы.
3. **Масштабирование одинаковых ролей:** Вы можете задействовать сразу трех кодеров для реализации трех разных фичей в разных директориях, или нескольких тестировщиков для покрытия тестами различных независимых модулей проекта одновременно.
4. **Нативная пакетная обработка:** Внутри параллельного пакета Maestro разделяет независимые задачи на слоты, что позволяет использовать ресурсы максимально эффективно без взаимных блокировок.

Такой подход значительно ускоряет разработку крупных многомодульных проектов и выполнение массовых операций (например, рефакторинг нескольких сервисов или добавление юнит-тестов ко многим компонентам одновременно).
